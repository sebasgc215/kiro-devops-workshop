# Contexto del Proyecto

## Descripción General

Este proyecto es un **workshop de DevOps** diseñado para aprender a usar Kiro como herramienta de desarrollo asistido por IA. El objetivo es practicar la creación y gestión de pipelines, infraestructura y flujos de trabajo modernos en un entorno controlado de aprendizaje.

## Stack Tecnológico

- **Runtime**: Node.js 20
- **Framework web**: Express
- **Testing**: Jest
- **Lenguaje**: JavaScript / TypeScript

## Deploy Target

El destino de despliegue es **AWS**, con dos opciones principales según el caso de uso:

- **AWS Lambda** — para funciones serverless, APIs ligeras o tareas event-driven
- **AWS ECS** — para servicios de larga duración, contenedores Docker o cargas de trabajo que requieren más control sobre la infraestructura

Al generar configuraciones de infraestructura o pipelines de despliegue, considerar ambas opciones y adaptar según el contexto del servicio.

## Preferencias de Comunicación

- Responder siempre **en español**
- Usar terminología técnica precisa, pero explicar conceptos cuando el contexto sea educativo
- Dado que es un workshop de aprendizaje, incluir comentarios explicativos en los archivos generados cuando aporten valor didáctico
